Sobre mi

Soy QA Tester junior con formación en testing manual, automatización, testing de APIs y metodologías ágiles, con un enfoque riguroso y orientado a la calidad del software y la experiencia del usuario.

Me caracterizo por mi capacidad analítica, anticipación de errores y comunicación clara con equipos técnicos, contribuyendo a mejorar procesos y reducir incidencias. Actualmente, busco seguir creciendo en el área de QA y avanzar hacia la automatización.

Proyectos

Testing de Pagina Web – QA Manual y Automatizado

Participé en el aseguramiento de calidad de una aplicación web dentro de un entorno ágil (Scrum), gestionando el ciclo completo de testing a lo largo de sprints semanales y participando en ceremonias como Sprint Planning, dailys y Sprint Review.

Diseñé y ejecuté casos de prueba basados en historias de usuario y criterios de aceptación, aplicando metodología BDD mediante escenarios en Gherkin, asegurando la trazabilidad entre requisitos, pruebas y resultados. Realicé pruebas funcionales, exploratorias y de regresión, reportando incidencias en Jira y colaborando en su seguimiento hasta resolución.

Contribuí a la documentación del plan de pruebas, resultados de ejecución y estado de calidad del producto en Confluence, así como en la preparación de releases al final de cada sprint.

Posteriormente, automatizé flujos críticos de regresión utilizando Playwright y Pytest con Python, integrándolos en un pipeline de integración continua con GitHub Actions para su ejecución automática periódica, mejorando la eficiencia del testing y la cobertura de calidad del producto.

Portafolio QA – Desarrollo Web y QA en Testing de APIs

Desarrollé un portafolio web utilizando HTML5, CSS3 y JavaScript, integrando un formulario conectado a una API generada con Supabase. Diseñé las tablas de la base de datos y realicé consultas SQL para insertar y consultar datos de prueba del formulario.

Realicé testing de APIs utilizando Postman, creando pruebas para los métodos GET, POST, PATCH y DELETE. Validé códigos de respuesta, autenticación, estructura y contenido de las respuestas, asegurando la correcta funcionalidad de la API.

Implementé un flujo de integración continua con GitHub Actions, configurando Newman para ejecutar automáticamente las pruebas de Postman con cada cambio en el repositorio.

Este proyecto refleja mi aprendizaje en desarrollo web, testing manual y de APIs, documentación de pruebas y trabajo colaborativo en entornos ágiles, además de demostrar el flujo completo desde desarrollo hasta despliegue en GitHub Pages.
